## 1. 배경 및 목적

### 1.1 배경

- 매일 TQQQ(나스닥100 3배 ETF)의 다음 거래일 **등락폭과 흐름을 예측**하고, 다음날 실측으로 **계속 튜닝**하고자 한다.
- 모멘텀/추세 추종은 연속(continuation) 구간은 잘 맞추지만, **추세가 꺾이는 변곡점(상승→하락, 하락→상승)** 에서 실패한다. 변곡점은 주로 **뉴스(촉매)** 가 유발한다.
- 나스닥100은 시총 상위 메가캡(NVDA·AAPL·MSFT·GOOGL·AMZN)이 좌우하므로, 이들 종목 뉴스 + 공통 매크로(전쟁·관세·경제지표·연준) 뉴스로 변곡을 포착하는 것이 핵심 가설이다.

### 1.2 목적

- 예측 산출물: **흐름(레짐) · 방향 · P20~P80 등락폭 밴드 · 신뢰도 · 근거 메타데이터(상위 3) · 변곡 표시**.
- 뉴스 분석 출력을 **티커별 지식그래프**로 누적·연결하여, 종목↔테마↔공통매크로 관계를 축적한다.
- 백테스트로 정확도를 정량 측정하고, 밴드 폭을 커버리지 60%에 캘리브레이션한다.

### 1.3 범위 제외 (Out of Scope)

- 실주문/자동매매 (예측·관측 전용).
- 일중(인트라데이) 예측 — 일 단위 종가 기준 예측만.
- TQQQ 외 종목 예측 (뉴스 그래프는 나스닥 top5 한정).

### 1.4 핵심 정직성 원칙 (검증으로 확정된 사실)

- **일간 방향/레짐 80%는 과적합 없이 도달 불가.** 공정한 혼합 90일(상승42·하락43) 기준 **방향 ~59% · 흐름 ~66%** 가 실력. 최근 불장 20~40일의 75~85%는 레짐 균일성에 의한 착시.
- 90일 중 **변곡일(모멘텀과 반대로 간 날) = 41%.** 이 변곡일을 뉴스로 잡는 것이 정확도 상향의 유일한 정직한 경로.
- 80% 를 정직하게 달성 가능한 지표는 **밴드 커버리지**(폭 조절) 또는 **주간(5일) 예측 horizon**.

│  [예측]  predict_core(features, news)                          │
│           ├ 밴드: 저가P20/고가P80 경험분위수 × 캘리브 폭        │

## 3. 기능 요구사항 (FR)

### FR-01 가격/기술 피처 (구현 완료)

- `price_service.get_overseas_daily(symbol, days)` — KIS 해외일봉 우선, Yahoo fallback. `{date,open,high,low,close,volume}` 오름차순.
- 라이브 실시간(KIS)과 동일 소스로 캘리브레이션 → 분할조정 드리프트 방지.
- 피처: 전일종가 기준 저/고/종가 상대수익률 분포, 모멘텀(5일), MA5/20/50, MA20·MA50 기울기, MA20 상회비율(above20), RSI, 실현변동성.
- **lookahead 차단**: i일 예측은 i-1일까지의 데이터만 사용.

### FR-02 뉴스 수집 (구현 완료)

- saveticker `/api/news/list` (텍스트 검색 API 없음 → 제목+본문 이름매칭).
- `collect_by_date(start_date)` 단일 패스로 전 기간을 날짜별 버킷팅 (티커 + 매크로).
- 티커: `TOP5_ALIASES`(KR/EN). 매크로: `MACRO_ALIASES`(전쟁/관세/경제지표/연준), 카테고리당 `_MACRO_CAP_PER_CAT=5`.
- 헤드라인별 리드 스니펫(`_LEAD_CAP=200`) 동봉 (제목+리드로 본문 맥락 보강).

### FR-03 뉴스 LLM 분석 — 모델 분기 (구현 완료)

- `_llm_json(prompt)` 백엔드 분기:
  - `NEWS_LLM=gemini` → Gemini API(`GEMINI_MODEL`), 429 지수백오프 재시도.
  - `NEWS_LLM=claude_cli` → `claude -p … --model {NEWS_CLAUDE_MODEL} --output-format json` (cwd=/tmp 로 프로젝트 컨텍스트 비용 회피, 코드펜스 제거).
- 출력 스키마: `{nasdaq_bias(-1~1), confidence, summary, per_ticker, events[], macro_events[]}`.
- 일당 1콜(하루치 헤드라인 배치). 실패(429/타임아웃)는 **캐시하지 않음** → 재시도 가능.
- 캐시: `data/news_bias_cache/{date}.json`.

### FR-04 티커별 지식그래프 (구현 완료)

- `ticker_graph.ingest_day(date, analysis)`:
  - `daily/{date}.json` — 1차 데이터(그날의 bias·티커이벤트·매크로이벤트).
  - `{TICKER}.json` — 종목별 이벤트 타임라인 + 테마 빈도.
  - `_macro.json` — 공통 카테고리별 상승/하락 이벤트 집계.
  - `_graph.json` — 노드(ticker/theme/macro) + 엣지(theme 언급, 공유테마 공출현, macro→MARKET).

### FR-05 예측 코어 (구현 완료)

- 밴드: 저가 P20 / 고가 P80 경험분위수 × 캘리브 폭 + 뉴스 bias 중앙 시프트.
- 방향: 모멘텀 + MA5/MA20 정렬 + 뉴스 bias.
- 흐름(레짐): MA 정배열/역배열 + MA20 기울기.
- 근거 메타데이터: |기여도| 상위 3 {factor, score, related} + 복수근거 공출현 연관 표시.

### FR-06 변곡점 포착 (로직 구현 완료, 검증 대기)

- 뉴스 bias 가 기술추세와 **강하게 반대**(`|bias| ≥ INFLECT_TH`, 현재 0.4)이면 방향 반전 예측.
  - 추세↑ & bias ≤ −임계 → `상승→하락`
  - 추세↓ & bias ≥ +임계 → `하락→상승`
- 채점: 실제반전일 중 예측비율(recall) / 예측반전 중 적중(precision).

### FR-07 백테스트 & 캘리브레이션 (구현 완료)

- `scripts/predict_backtest.py` — `--days --lookback --width --news --rhorizon`.
- 밴드 폭 그리드(0.3~2.0) 탐색 → 커버리지 60% 최근접 자동 캘리브.
- 출력: 방향적중 · 흐름적중(전방 rhorizon일 추세) · 밴드커버 · 폭편향 · 변곡 recall/precision + 근거 메타 샘플.

### FR-08 라이브 예측 & 자가튜닝 (미구현 — 남은 작업)

- 매 거래일 예측 발행 → ledger 기록 → 다음날 실측 채점 → 최근 N일 성적·편향을 프롬프트에 주입(LLM 피드백-인-컨텍스트).
- 상세는 §8 남은 작업 참조.

## 5. 외부 인터페이스 명세

### 5.1 KIS 해외일봉

- `GET /uapi/overseas-price/v1/quotations/dailyprice` (tr_id `HHDFS76240000`)
- params: `AUTH="", EXCD, SYMB, GUBN=0(일), BYMD(기준일), MODP=1(수정주가)`
- EXCD 디스커버리: NAS→NYS→AMS, 성공값 캐시. 페이지네이션(1콜 ~100봉).

### 5.2 saveticker 뉴스

- `GET https://api.saveticker.com/api/news/list?page=&page_size=&sort=created_at_desc`
- 응답: `{news_list[], total_count, …}`. 아이템: `id, title, content(리드~80자), created_at(KST), tag_names …`
- **검색(q/keyword) 파라미터 무시됨** → 클라이언트 이름매칭 필수.
- 상세 본문: `/api/news/detail/{id}` (속보는 본문 없음 — 사용 안 함).

### 5.3 LLM

- Gemini: `POST generativelanguage.googleapis.com/v1beta/models/{model}:generateContent` (`responseMimeType=application/json`, 2.5-*는 `thinkingBudget=0`).
- Claude CLI: `claude -p {prompt} --model {haiku|sonnet|opus} --output-format json` → `.result`(코드펜스 제거 후 파싱).

## 부록 B. 용어

- **흐름(레짐)**: 다일 추세 방향(상승흐름/하락흐름/중립). 1일 방향과 별개.
- **P20/P80 밴드**: 장중 저가 20%분위 ~ 고가 80%분위. 양 꼬리 20%씩 제외한 ~60% 커버리지 밴드.
- **변곡점(inflection)**: 모멘텀 추세와 반대로 움직이는 반전일. 뉴스 촉매가 주 원인.
- **LLM 피드백-인-컨텍스트**: 최근 예측 성적·편향을 다음 예측 프롬프트에 주입해 자가보정하는 튜닝 방식.
- **nasdaq_bias**: 그날 뉴스의 다음 거래일 지수 방향 종합 점수 (−1 강한하방 ~ +1 강한상방).
